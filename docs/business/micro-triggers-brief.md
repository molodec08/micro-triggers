# Бриф: "Микро-триггеры внимания" — бизнес-часть

См. также: [техническую часть](../technical/micro-triggers-tech.md),
[разведку конкурентов](competitor-research.md).

## Контекст и цель
Небольшое платное приложение для Shopify App Store (запасной/альтернативный вариант
к age-verification приложению).

## Идея и дифференциатор
Набор лёгких "микро-триггеров внимания" для удержания посетителя в момент,
когда он собирается уйти с товаром в корзине.

Рынок exit-intent/попапов сильно занят тяжёлыми игроками (Privy, OptinMonk,
Justuno) — $20-40/мес, с капами по трафику (1000-3000 просмотров на бесплатном/
базовом тарифе), богатый конструктор шаблонов, email/SMS-интеграции.

Дифференциатор: НЕ конкурировать функциональностью, а сделать нарочито лёгкую,
дешёвую альтернативу без ограничений по трафику:
- Флэт-цена **$9/мес, без капов по просмотрам страниц**
- Минимальный вес бандла (кило-, не мегабайты) — не тормозит магазин
- Простота вместо конструктора шаблонов

> Обновление после разведки конкурентов (см. [competitor-research.md](competitor-research.md)):
> ниша "дёшево и без капов" уже занята (Section Store, StoreYa Exit Pop, Before
> You Go). Рекомендация — сместить УТП в сторону "3 лёгких триггера в одном
> приложении", а не только цены.

## MVP-функции (v1)
1. **Мигающая вкладка браузера** при уходе с вкладки, если в корзине есть товар
2. **Минималистичный exit-intent попап** — только текст + промокод, без конструктора шаблонов
3. **Звуковой сигнал** при добавлении в корзину / на checkout

Сознательно НЕ в MVP: браузерные push-уведомления — сложнее технически.
Кандидат на v2 как апгрейд более дорогого тарифа.

## Roadmap (недели) — короче, чем age-verification, т.к. вся логика фронтовая
1. Разведка конкурентов, фиксация позиционирования "лёгкий и без ограничений",
   Partner-аккаунт, dev store, CLI-скелет
2. Backend: OAuth, минимальная модель настроек в Postgres
3. Storefront: реализация трёх триггеров
4. Admin UI (тумблеры, превью) + Billing API ($9/мес, без капов по трафику)
5. QA: кросс-темы, проверка с блокировщиками рекламы, подготовка листинга
   с акцентом на "лёгкий и дешёвый" месседж
6. Сабмит на ревью (2-6 недель) + продвижение в Partner Slack/форумах с
   позиционированием против тяжёлых конкурентов

## После запуска
Если пойдёт — v2 может добавить браузерные push-уведомления как апгрейд более
дорогого тарифа, раз там уже отдельная техническая сложность (service worker),
которая оправдывает более высокую цену.

## Альтернативная идея (основной вариант, более приоритетный)
Age-verification приложение для магазинов с товарами 18+/21+ — см. отдельный
бриф `age-verification-brief.md` (пока не создан на диске). Оценивался как более
жизнеспособный как самостоятельный продукт, эта идея — быстрее в разработке,
но потенциал MRR ниже.
